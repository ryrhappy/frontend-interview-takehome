# 前端技术测试项目

## 项目概述

这是一个使用 Next.js 和 TypeScript 构建的简化版物业管理系统（PMS）UI。

**该应用存在一些性能和架构问题。** 你的任务是：

1. 尽可能多地发现并修复问题
2. 在 `DECISIONS.md` 中记录你的发现（见下方说明）
3. 使用git管理你的代码，解压后首先git初始化你的项目
4. 将你发现的问题，以commit为单位分别提交
5. 最终推入一个你自己的公开仓库，把地址在群里分享出来即可
6. 如果对本次试题有任何问题，也可以在群里提问

---

## 开始使用

```bash
npm install
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)。

---

## 应用功能

- **预订页面 (`/`)** — 一个可滚动的房间/预订网格，显示 30 个房间在 30 天日期范围内的预订情况。点击任何预订条可打开详情抽屉。
- **消息页面 (`/messages`)** — 客户支持工单列表。点击工单查看详情；URL 会更新为选中的工单 ID。

## 交付物

1. **可运行的代码** — 你的修复已提交，可通过 `npm run dev` 运行
2. **项目根目录下的 `DECISIONS.md`** — 一份简短文档（最多 1 页），包含：
   - 你发现的问题

## 注意事项

- 不要安装额外的主要依赖（不要使用 Redux、Zustand、React Query 等）
- 数据获取请继续使用 SWR
- TypeScript 类型已在 `types/index.ts` 中提供 — 你可以扩展它们但不要删除
- `pages/api/` 中的模拟 API 路由返回真实数据，并模拟网络延迟
- 组件中的 `console.log` 调用是有意为之的 — 在理解它们的作用之前不要删除

---

## 评估标准

我们不是在寻找完美的代码。我们关注的是：

- **问题发现能力** — 你是否找到了问题？
- **推理质量** — 你能否解释 _为什么_ 某些东西是错的，而不仅仅是指出它是错的？
- **修复质量** — 你的修复是否正确且恰当？
- **诚实度** — 你是否清楚地说明了你发现但未修复的问题，以及原因？
